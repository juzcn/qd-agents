# qd-agents

意图驱动的上下文隔离多Agent系统

## 架构概述

本项目实现了一个三模块架构的多Agent系统：

```
用户输入
    ↓
┌─────────────────────────────────────────┐
│          会话Agent (有状态)             │
│  输入：会话历史 + 长期记忆 + 人格/规则  │
│  输出：自然语言回复 + 意图对象(结构化)  │
└─────────────────────────────────────────┘
    │                                ↑
    │ 意图对象                   │ 执行结果(作为新记忆)
    ↓                                │
┌─────────────────────────────────────────┐
│          工具Agent (无状态)             │
│  输入：意图对象 + 工具列表(Schema)      │
│  输出：工具调用(tool_calls)             │
└─────────────────────────────────────────┘
    │
    │ 工具名+参数
    ↓
┌─────────────────────────────────────────┐
│        执行层 (非Agent)                  │
│  输入：工具调用指令                      │
│  输出：执行结果 / 需要用户交互的信号     │
└─────────────────────────────────────────┘
```

## 核心原则

- **上下文最小化**：只有会话Agent可访问会话历史和长期记忆
- **意图作为唯一桥梁**：会话Agent对外输出"完整意图"
- **无状态工具链**：工具Agent和执行层不保留任何跨请求状态
- **自然语言输出**：会话Agent输出自然语言回复 + 结构化意图

## 项目结构

```
qd-agents/
├── src/qd_agents/
│   ├── __init__.py
│   ├── agents/              # Agent模块
│   │   ├── __init__.py
│   │   ├── session_agent.py # 会话Agent
│   │   └── tool_agent.py    # 工具Agent
│   ├── execution/           # 执行层
│   │   ├── __init__.py
│   │   └── executor.py      # 工具执行器
│   ├── intent/              # 意图对象
│   │   ├── __init__.py
│   │   ├── schema.py        # Intent Schema定义
│   │   └── builder.py       # 意图构建器
│   ├── memory/              # 记忆模块
│   │   ├── __init__.py
│   │   ├── short_term.py    # 短期记忆
│   │   └── long_term.py     # 长期记忆
│   ├── orchestrator.py      # 编排器
│   └── cli.py               # 命令行入口
├── pyproject.toml
└── README.md
```

## 安装

使用 `uv` 安装：

```bash
uv sync
```

## 开发

```bash
# 安装依赖
uv sync --dev

# 运行类型检查
uv run mypy src/qd_agents

# 运行测试
uv run pytest
```
